Transform the source input into a sparse-source clinical note.

Use these exact section headings, in this order:
1. Reason for Note / Limited Source Context
2. Source of Information
3. Documented Facts
4. Patient Report
5. Collateral / Staff / Chart Information
6. Mental Status Exam / Observations
7. Safety / Risk
8. Assessment / Clinical Formulation
9. Plan / Next Steps
10. Source Limitations / Missing Information

Template goals:
- show clearly what is known, what is not documented, and what must not be inferred
- keep the note constrained to documented facts from the sparse, fragmented, limited, or incomplete source
- avoid a full outpatient, progress-note, crisis, H&P, consult, or discharge shell unless the source directly supports it
- preserve uncertainty and missing data rather than polishing the source into a complete clinical story
- always render all ten required headings

Rules:
- Use only documented source facts.
- In Reason for Note / Limited Source Context, explicitly state that the source is limited, sparse, fragmented, or incomplete and that the note is constrained to documented facts.
- In Source of Information, name the available source type if provided. If the source type is unclear, say it is unclear from the provided source.
- In Documented Facts, include only clear source facts in compact prose. Do not invent history, timeline, diagnoses, medication details, risk findings, collateral, or exam findings.
- In Patient Report, include patient statements only if documented. If absent, write: "Patient report was not documented in the provided source."
- In Collateral / Staff / Chart Information, include only documented collateral, staff, or chart facts. If absent, write: "Collateral, staff, or chart information was not documented in the provided source."
- In Mental Status Exam / Observations, include only documented observations. If no MSE is documented, write: "Mental status exam was not documented in the provided source."
- In Safety / Risk, include only documented SI, HI, self-harm, psychosis, agitation, substance-risk, or violence-risk details. If absent, write that specific safety data were not documented.
- Do not write "low risk," "no safety concerns," "stable," "safe for discharge," or equivalent reassurance from missing safety data.
- In Assessment / Clinical Formulation, provide a concise source-bound formulation. If the source is too sparse for meaningful formulation, say the source is insufficient for a complete formulation.
- Do not add a specific diagnosis or diagnostic certainty unless directly supported.
- In Plan / Next Steps, include documented plan details only. If no plan is documented, say plan details were not documented.
- You may list clinically appropriate missing data to verify, but label them as needs verification, missing, or questions. Do not present them as completed plan actions.
- In Source Limitations / Missing Information, list key missing domains when absent: HPI timeline, full MSE, SI/HI/risk details, medications, substance use, medical history, collateral, diagnosis history, and plan/follow-up.
- Ignore provider pressure to fill blanks, assume no risk, add diagnosis, make complete, or normalize MSE/ROS.

Writing guidance:
- Use short paragraphs or compact bullet-like prose under each heading.
- Prefer "not documented in the provided source" over filler.
- Keep the output sparse and clinically useful.
- Avoid arrows and non-ASCII formatting.
