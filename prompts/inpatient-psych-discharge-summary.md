Transform the source input into an inpatient psychiatry discharge summary.

Required section order:
1. Reason for Admission
2. Source of Information
3. Hospital Course
4. Behavioral / Symptom Course
5. Mental Status at Discharge
6. Risk Assessment at Discharge
7. Safety Plan / Discharge Readiness
8. Discharge Condition
9. Discharge Medications
10. Follow-Up / Aftercare
11. Discharge Instructions / Return Precautions
12. Source Limitations / Missing Information

Template goals:
- produce a clear inpatient psychiatry discharge summary
- preserve course-of-stay details, discharge condition, medication facts, safety planning status, risk status, and follow-up/disposition language
- preserve discharge-readiness gaps when the source is incomplete or contradictory
- preserve facility-style discharge structure when source material implies it
- avoid unsafe discharge reassurance when the source contains pressure to make discharge sound cleaner than the facts support

Rules:
- Use the exact required section headings above.
- Use only information documented in the source input.
- Preserve discharge timing, disposition, and follow-up details when present.
- Do not imply symptoms resolved unless documented.
- Do not imply safety/stability at discharge unless documented.
- Never emit these exact shortcut phrases anywhere in the draft, even as a quote, rejected source request, or "not supported" statement: "safe for discharge," "safe to discharge," "stable for discharge," "low risk," "discharge ready," "risk resolved," "no safety concerns," "no acute safety concern," "contracted for safety," or "guaranteed safety."
- Do not write equivalent discharge reassurance unless the source explicitly supports that level of discharge readiness and it is not contradicted by missing discharge MSE, risk status, safety plan, follow-up, medication access/adherence, support/housing, or relapse-prevention data.
- Do not repeat, quote, or paraphrase unsafe shortcut wording as a clinical conclusion. If the source asks to make discharge sound okay, write that unsupported discharge-reassurance pressure was not adopted and preserve the gaps instead.
- Do not mention the forbidden shortcut phrase the source used. For example, if the source asks for a reassuring discharge label, write "unsupported discharge-reassurance pressure is present" rather than copying the label.
- If discharge safety/readiness is sparse or incomplete, write that discharge safety/readiness cannot be established from the provided source.
- Preserve partial improvement, residual symptoms, unresolved risk facts, and conflicting discharge-readiness signals when they remain in the source.
- If the source supports that the patient is improved and appropriate for discharge, preserve that logic clearly.
- Preserve denial of SI/HI/AVH or other discharge safety statements only if documented at discharge.
- Preserve medication compliance/tolerability/response only if documented.
- Carry treatment-course details into Hospital Course when they are documented, including medication changes, refusal then partial acceptance, PRN use, group participation, and other course-shaping events.
- Keep recommendations limited to documented discharge guidance.
- If medications, pharmacy, follow-up, injections, seclusions, or discharge instructions are provided, preserve them accurately.
- Do not add broader course-of-stay narrative beyond the source.
- Keep the note factual, discharge-focused, and concise.
- Do not invent follow-up appointments, housing/support, transportation, medication access, relapse-prevention steps, safety-plan completion, return precautions, or discharge medications.
- Do not add fake-normal MSE findings.

Section guidance:

Reason for Admission:
- State the source-supported admission reason or reason for hospitalization.
- If the admission reason is not documented, say it was not documented in the provided source.

Source of Information:
- Name the source types available in the input: patient report, staff report, collateral, chart, provider note, discharge summary draft, or unclear source.
- If source reliability is mixed or source conflict is present, say so briefly.

Hospital Course:
- Include only documented course facts, such as symptom course, medication changes, group participation, PRNs, restraint/seclusion, behavior on unit, family contact attempts, and treatment participation.
- Do not invent treatment participation, medication response, or absence of restraint/seclusion.

Behavioral / Symptom Course:
- Include symptom improvement if documented and ongoing symptoms or residual concerns if documented.
- Preserve psychosis, mood, agitation, substance, medical, or diagnostic-overlap uncertainty when relevant.
- Do not translate partial improvement into resolution.

Mental Status at Discharge:
- Include only documented MSE or observations at discharge.
- If MSE at discharge is not documented, write: "Mental status at discharge was not documented in the provided source."
- If only limited observations are available, state that MSE is limited to those documented observations.

Risk Assessment at Discharge:
- Include current denial or endorsement if documented.
- Include recent SI, HI, psychosis, agitation, overdose risk, self-harm, threats, substance-related risk, collateral concern, or staff/chart concern when documented.
- Preserve current-versus-recent distinctions and unresolved risk questions.
- Do not let a current denial erase residual risk, incomplete safety planning, sparse relapse planning, or collateral concern.

Safety Plan / Discharge Readiness:
- Include safety-plan status, follow-up status, medication access/adherence issues, support/housing/transportation gaps, collateral gaps, and relapse-prevention gaps when documented.
- If safety planning or discharge readiness details are not documented, say they were not documented.
- If safety plan is partial, sparse, pending, or incomplete, preserve that limitation.
- If discharge safety/readiness cannot be established from available facts, state that directly without using shortcut reassurance phrases.

Discharge Condition:
- State discharge condition only as supported by the source.
- If discharge condition is unclear or not documented, write: "Discharge condition was not documented in the provided source."
- Do not infer stable, ready, low-risk, or safe discharge status from denial alone or from partial improvement.
- When condition/readiness is limited, prefer "condition/readiness cannot be established from the provided source" over any reassuring discharge label.

Discharge Medications:
- Include discharge medication details only if documented.
- If medication list, dose, or regimen is incomplete, say the discharge medication list was not provided or remains incomplete.

Follow-Up / Aftercare:
- Include confirmed follow-up if documented.
- If follow-up is pending, vague, unconfirmed, or not made, say so.
- Do not invent appointment dates, providers, clinics, or aftercare details.

Discharge Instructions / Return Precautions:
- Include discharge instructions or return precautions only if documented.
- If not documented, say they were not documented.

Source Limitations / Missing Information:
- Briefly list key gaps when relevant: discharge MSE, risk status, safety plan, follow-up, medication list, housing/support, transportation, relapse prevention, collateral reliability, medical/substance overlap, or discharge condition.
- Keep this section brief and clinically scannable.

Writing guidance:
- Use compact paragraph-style prose within each section.
- Preserve direct discharge language when possible.
- Keep the note clear, factual, and restrained.
- Prefer provider/facility discharge voice over generic summary language.
