## How to Use Veranote: From Messy Notes to Final Draft

### 1. What Veranote does
Veranote helps turn rough clinical source material into a cleaner draft note.

It can help you:
- build a note from messy source text
- clean up wording
- preserve contradictions and uncertainty
- improve chart-ready language
- answer medication reference questions with verification reminders
- give provider-review switching frameworks for medication transitions

### 2. What Veranote does not do
Veranote does not replace clinician judgment.

It should not:
- invent missing facts
- make you the wrong clinician of record
- sign or submit the note for you
- replace prescribing references
- create final medication orders
- create final cross-taper or switching orders

You remain responsible for final review.

### 3. Step 1: Choose your note type
Start by picking the note type that matches the visit.

Examples:
- inpatient psych initial adult evaluation
- inpatient psych progress note
- inpatient psych discharge summary
- outpatient psych follow-up
- psychiatric crisis note
- medical consultation note

If you are unsure, pick the note type closest to the final note you need.

### 4. Step 2: Paste your source information
Paste the rough facts you already have.

This can include:
- your own rough bullets
- copied chart text
- collateral
- patient quotes
- transcript excerpts
- labs, vitals, and medication details

The best results come from giving Veranote the messy facts, not polished language.

Raw bullets, partial chronology, and incomplete source are okay.

### 5. Step 3: Generate your draft
Once the source is loaded, generate the draft.

Veranote will try to:
- keep the note source-close
- preserve uncertainty
- avoid inventing findings
- keep missing details marked as not documented when needed

### 6. Step 4: Review the draft
Read the draft before copying anything into the chart.

Check:
- dates and timeline
- risk wording
- medication details
- contradictions
- collateral vs patient report
- whether anything important is missing

If information was not in your source, Veranote may say it was not documented.

That is usually safer than filling in blanks.

### 7. Step 5: Ask Vera for help if needed
Use Vera when you want help with wording, not when you want the system to take over judgment.

Good uses:
- “word this better”
- “make this chart-ready”
- “preserve the contradiction”
- “how do I document this more clearly”
- “give me chart-ready wording instead”

Iterative refinement works well:
- initial response
- correction
- final wording

### 8. Step 6: Copy or save your final note
When the note looks right, use the copy or export option that fits your workflow.

If your EHR accepts a full note, use the main copy action first.

If your EHR splits the note into sections, use the field-level or plain-text option only if needed.

Save the draft if you need to come back later.

### 9. Medication/reference questions
You can ask Vera medication reference questions such as:
- starting dose
- usual range
- monitoring
- interaction concerns
- brand/generic lookup
- medication documentation wording
- switching or cross-taper frameworks

Medication and switching answers must be verified with current references.

Vera is not a licensed drug database.

Switching answers are provider-review frameworks, not patient-specific orders.

### 10. Safety reminders
- You are responsible for the final note.
- Vera should not invent missing facts.
- If the source is thin, the output may stay cautious.
- Medication and switching answers require current prescribing or interaction-reference verification.
- If the draft sounds more certain than the source, revise it before use.

### 11. Tips for better results
- Paste the real working facts, even if they are messy.
- Keep patient report, collateral, and objective data separate when they conflict.
- Include the risk facts you do not want lost.
- Include medication refusal, side effects, adherence, and follow-up gaps when they matter.
- If the first draft is close but not right, ask Vera to revise the exact sentence or section.

### 12. What to do if Vera gets something wrong
Stop and correct it before using the note.

You can:
- ask Vera for a more source-close rewrite
- ask for chart-ready wording
- add the missing fact and regenerate
- remove language that overstates certainty
- document the failure for follow-up review

If Vera misses a key fact, invents something, or gives wording that is not usable, treat that as a system miss and report it so it can be turned into a future regression case.
