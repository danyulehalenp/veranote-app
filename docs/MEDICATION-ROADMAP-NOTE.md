# Medication Roadmap Note

## Now
Keep medication work inside the current trust stack:
- prompt rules
- review warnings
- eval cases
- explicit anti-invention guidance

## Bring the larger medication subsystem back only when
- medication conflict cases still fail after the current guardrails mature
- review UX can already show evidence clearly for medication statements
- the team has repeated proof that medication complexity is the next real bottleneck
- there is a product reason to support structured medication state across encounters or note types

## Before that, do these first
1. tighten medication-specific eval coverage
2. improve evidence-linked review for medication statements
3. confirm which medication failures are still recurring
4. only then decide whether lightweight structured extraction is enough or a true subsystem is warranted
