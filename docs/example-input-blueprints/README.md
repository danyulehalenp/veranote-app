# Veranote Example Input Blueprints

These blueprints turn the mined ChatGPT founder-history signal into reusable Veranote product assets.

They are intentionally:

- de-identified
- pattern-based
- psych-first
- designed for product shaping, eval creation, and trusted-provider beta preparation

They are **not** raw patient cases and should not be treated as representative clinician consensus.
They reflect a strong founder-workflow dataset and should later be balanced by provider-beta feedback.

## How To Use These

- product design:
  - decide which inputs Veranote should accept comfortably
  - decide which trust cues must appear during review
- eval creation:
  - derive note-generation and review cases from these input shapes
  - test what the model preserves versus overstates
- provider beta:
  - show beta users realistic example patterns
  - compare founder workflow to other provider workflows

## Working Rule

These blueprints are meant to answer:

`What kind of raw clinician input should Veranote handle gracefully?`

not:

`What exact patient text should be reused?`
